## 2. 실행 검증 결과

`main.py`는 실행 가능한 상태로 확인되었다. 다만 최초 확인 시 단어 빈도 계산 부분에서 `torch.bincount()` 결과인 `word_counts`가 아니라 원본 ID 텐서인 `word_counts_tensor`를 참조하고 있었다.

이 문제 때문에 실제 빈도가 아닌 단어 ID 값이 빈도처럼 출력되어 `오른 78`, `토너먼트 78`처럼 기사 내용과 맞지 않는 결과가 나왔다. 해당 부분은 아래처럼 수정했다.

```python
word_freq = {
    vocab[i]: int(word_counts[i].item())
    for i in range(len(vocab))
}
```

수정 후에는 단어 빈도가 정상적으로 계산되었다.

## 3. 전처리 결과

정규표현식을 사용하여 한글과 공백만 남기고 숫자, 영어, 특수문자를 제거했다.

```python
cleaned_text = re.sub(r"[^ㄱ-ㅎㅏ-ㅣ가-힣\s]", " ", text)
cleaned_text = re.sub(r"\s+", " ", cleaned_text)
```

이 과정에서 `32강`, `2026년`, `38%`, `0대0` 같은 숫자 정보는 제거된다. 그래서 최종 분석은 경기 수치보다는 기사에 반복적으로 등장하는 인물, 국가, 팀, 상황 중심으로 이루어진다.

## 4. 명사 추출 결과

Okt의 `nouns()`를 사용해 명사를 추출했고, 길이가 1 이하인 단어는 제거했다.

- 추출 명사 수: 134개
- 고유 단어 수: 81개
- 불용어 제거 방식: 현재는 `len(word) > 1` 기준만 적용

현재 방식은 간단한 필터링에는 적절하지만, `디애`, `슬레`, `위인`, `강전`처럼 형태소 분석 과정에서 어색하게 분리된 단어가 일부 남는다. 더 깔끔한 분석을 원하면 별도 불용어 리스트를 추가하는 것이 좋다.

## 6. 결과 해석

가장 많이 등장한 단어는 `멕시코`로 11회 출현했다. 기사 전체가 멕시코의 32강 대진 가능성과 유리한 토너먼트 경로를 중심으로 구성되어 있기 때문에 가장 핵심적인 키워드로 볼 수 있다.

`스코틀랜드`, `카보베르데`, `사우디`는 멕시코의 예상 상대 후보로 반복 등장했다. 이 세 단어가 상위권에 있는 것은 기사 주제가 멕시코의 다음 상대 분석이라는 점과 잘 맞는다.

`한국`은 4회 등장하며 기사 후반부의 비교 대상으로 쓰였다. 한국은 조 1위 실패와 향후 대진 부담을 설명하는 맥락에서 등장하므로, 기사 구조상 멕시코와 대비되는 보조 핵심어로 해석된다.

`꽃길`, `승점`, `월드컵`, `상대` 같은 단어는 기사 제목과 본문 주제를 보조한다. 특히 `꽃길`은 멕시코의 유리한 대진을 상징적으로 표현하는 단어다.

## 7. 코드 품질 점검

현재 코드는 전체 과제 흐름을 대부분 충족한다.

- 텍스트 정제 단계가 분리되어 있다.
- Okt 형태소 분석 결과를 확인할 수 있다.
- 명사 기반 분석으로 워드클라우드에 적합한 입력을 만든다.
- PyTorch Tensor와 `torch.bincount()`를 사용하여 빈도를 계산한다.
- pandas Series로 상위 단어를 확인한다.
- WordCloud 생성 코드가 포함되어 있다.

다만 개선하면 좋은 부분도 있다.

- `plt.show()`만 사용하면 자동 검증 환경에서는 이미지 파일이 남지 않는다. `plt.savefig("wordcloud.png")`를 추가하면 결과물을 재확인하기 쉽다.
- 현재 불용어 제거는 길이 기준만 적용되어 있다. `디애`, `슬레`, `위인` 같은 단어를 제거하려면 별도 불용어 리스트가 필요하다.
- 숫자를 모두 제거하기 때문에 `32강`, `1위`, `승점 4` 같은 스포츠 기사 핵심 정보가 사라진다. 숫자 정보도 분석 대상에 포함하려면 정규식을 조정해야 한다.
